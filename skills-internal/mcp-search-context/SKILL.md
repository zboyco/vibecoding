# search_context 专用技能

## 使用目标

- 在需要查找、搜索、定位代码时，优先使用 `mcp search_context`。
- 减少盲搜与误改，先定位上下文，再进入修改。

## 核心规则

- 搜索代码时优先使用 `mcp search_context`，不要先做全量人工遍历。
- 查询描述先写“行为/流程目标”，再补充关键字。
- 对检索结果做二次核验（文件位置、职责、调用链）后再修改代码。

## 查询建议

- 推荐结构：`自然语言目标 + 关键词`。
- 示例：
  - “定位鉴权失败后的回退逻辑。关键词：auth fallback middleware”
  - “查找任务完成后触发通知的实现。关键词：notify taskName”

## 结果校验

- 至少确认：目标文件是否正确、是否属于当前改动范围、是否存在关联调用点。
- 若首次检索结果不充分，调整关键词再次检索，而不是直接拍脑袋改动。
