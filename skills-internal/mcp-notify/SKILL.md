# notify 专用技能

## 使用目标

- 统一 `mcp notify` 的触发时机与调用方式。
- 保证关键节点通知稳定、标题清晰、顺序一致。

## 触发时机

- 任务阶段完成后需要用户确认时。
- 任务执行中遇到关键决策点需要反馈时。
- 任务整体完成准备汇报时。

## 核心规则

- 调用 `mcp notify` 时必须提供 `taskName`。
- `taskName` 必须简短、准确、能一眼识别当前任务主题。
- 严格遵循顺序：`执行任务 -> 通知用户 -> 总结汇报`。

## 标题建议

- `需求理解确认`
- `方案选择待确认`
- `计划待审批`
- `执行进度更新`
- `任务交付总结`
